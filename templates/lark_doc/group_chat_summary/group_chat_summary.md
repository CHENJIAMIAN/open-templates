# SKILL: group_chat_summary

基于飞书群聊的**智能摘要与洞察**生成能力。从指定群聊/时间范围的消息中提取关键信息，识别决策、行动项、风险，生成**飞书云文档结构化摘要**。

---

## 1) Overview

### 触发方式（Triggers）
- "帮我总结一下 XX 群的聊天记录"
- "把这个群最近一周的讨论整理成摘要"
- "总结今天群里的重要决策和待办事项"
- "帮我追踪 XX 项目群里关于 YY 话题的讨论"

### 输入（Inputs）
- **必选**：群聊标识（群名/群ID/群链接）
- **可选**：
  - 时间范围（默认：过去 24 小时）
  - 关注主题/关键词
  - 输出粒度（简洁版/标准版/详细版）

### 核心输出（Deliverables）
- **飞书云文档群聊摘要**：结构化、可追溯、行动项可执行

### 质量标准
- **完整性**：不遗漏关键决策与行动项
- **准确性**：可回溯到原始消息
- **可操作**：行动项有 Owner / 截止时间

## 3) Module A — 消息获取（主 Agent 执行）

> **核心原则**：先拉取原始消息，再做时间过滤、去噪、分段。

### A1. 检索工具组合

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| `mcp__feishu_im__*` | 获取群聊消息 | 飞书群聊消息拉取 |
| `knowledge_answer` | 补充相关上下文 | 历史决策、项目背景 |
| `feishu_doc_search` | 检索关联文档 | 群内分享的云文档 |

### A2. 时间窗口定义

**默认**：过去 24 小时（用户时区）

**支持自定义**：
- "今天" / "本周" / "最近 3 天"
- 指定起止时间

**写入 `draft.md`**：
```markdown
## 群聊信息
- 群名：{群名}
- 统计范围：YYYY-MM-DD HH:MM ~ YYYY-MM-DD HH:MM (UTC+8)
- 消息总数：XXX 条
- 参与人数：XX 人
```

### A3. 消息预处理

**去噪规则**：
- 纯表情/贴纸：标记但不纳入主分析
- 系统消息（入群/退群）：单独记录或忽略
- 重复消息：去重，保留首次
- 纯闲聊（问候/感谢）：降权处理

**消息分段**：
- 按时间间隔（>30分钟）自动切分
- 按话题切换辅助分段
- 每段标注：起止时间、参与人、主题标签

**写入 `draft.md`**：
```markdown
## 消息分段

### 段落1：{主题标签}
- 时间：HH:MM ~ HH:MM
- 参与人：@A, @B, @C
- 消息数：XX 条
- 核心内容摘要：...
```

## 4) Module B — 结构化提取（主 Agent 执行）

> **核心原则**：提取"决策、行动、风险、资源"四类关键信息。

### B1. 核心议题识别

**识别方法**：
- 从讨论段落中提取主要话题（3-7 个）
- 按讨论热度排序

**写入 `draft.md`**：
```markdown
## 核心议题

| 序号 | 议题 | 热度 | 时间段 | 参与人 |
|------|------|------|--------|--------|
| 1 | XX 项目排期 | 高 | 10:00-11:30 | @A, @B |
| 2 | XX 技术方案 | 中 | 14:00-14:30 | @C, @D |
```

---

### B2. 关键决策提取

**识别信号**：
- "决定..."、"确认..."、"就这样定了"、"同意"、"通过"
- 投票结果
- 领导/关键人拍板

**写入 `draft.md`**：
```markdown
## 关键决策

### 决策1：{决策内容}
- **决策时间**：HH:MM
- **决策人**：@XX
- **参与人**：@A, @B
- **背景**：...
- **原始消息**："{引用原文}"
```

---

### B3. 行动项提取

**识别信号**：
- "@XX 你来负责..."、"XX 跟进一下"
- "我来做..."、"我负责..."
- 截止时间提及："周五前"、"明天"

**写入 `draft.md`**：
```markdown
## 行动项

| 任务 | 负责人 | 截止时间 | 状态 | 原始消息 |
|------|--------|---------|------|---------|
| 完成 XX 方案 | @张三 | 01-20 | 待开始 | "张三你来写方案" |
| 跟进 XX 审批 | @李四 | 01-18 | 进行中 | "李四催一下审批" |
```

---

### B4. 风险与问题提取

**识别信号**：
- "问题是..."、"风险在于..."、"卡住了"、"阻塞"
- "谁能帮忙..."、"这个怎么解决"

**写入 `draft.md`**：
```markdown
## 风险与问题

### 问题1：{问题描述}
- **严重程度**：高/中/低
- **提出人**：@XX
- **提出时间**：HH:MM
- **状态**：待解决/已解决
- **解决方案**：...（如有）
```

---

### B5. 重要链接与资源

**提取类型**：
- 飞书文档链接
- GitHub/GitLab 链接
- 外部链接（文章/工具）
- 文件附件

**写入 `draft.md`**：
```markdown
## 重要链接

| 资源 | 类型 | 分享人 | 时间 | 关联议题 |
|------|------|--------|------|---------|
| XX技术方案 | 飞书文档 | @张三 | HH:MM | 议题1 |
| XX代码库 | GitHub | @李四 | HH:MM | 议题2 |
```

## 5) Module C — 调用 Writer Agent 生成摘要

### 任务分发

主 Agent 完成提取后，调用 `task` 工具分发给 writer agent：

```json
{
  "description": "生成群聊摘要云文档",
  "prompt": "基于 /home/workspace/draft.md 中的群聊分析结果，生成一份飞书云文档群聊摘要。\n\n要求：\n1. TL;DR 放在最前面（3-5 条要点）\n2. 决策和行动项使用表格展示\n3. 行动项必须有负责人和截止时间\n4. 使用 Callout 突出重要决策和风险\n\n摘要模板见下方。",
  "subagent_type": "writer"
}
```

## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 遗漏关键决策 | 信号词不全 | 扩大决策识别信号词 |
| 行动项无 Owner | 未明确分配 | 标注"待分配"，列入待跟进 |
| 噪音过多 | 闲聊未过滤 | 加强去噪规则 |
| 时间范围错误 | 时区问题 | 明确标注时区 |
| 摘要过于笼统 | 缺少具体信息 | 引用原始表述、补充时间人名 |

## 10) 扩展场景

### 10.1 主题追踪模式
- 用户指定关键词/话题
- 跨时间段追踪该话题演进
- 输出：话题时间线 + 各阶段结论

### 10.2 多群聚合
- 多个相关群的讨论汇总
- 识别跨群关联话题
- 全局行动项汇总表

### 10.3 周期性摘要
- 每日/每周自动生成
- 与历史摘要对比
- 追踪长期议题进展
