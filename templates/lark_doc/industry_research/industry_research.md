# SKILL: industry_research

面向特定行业/赛道的**系统性深度研究**能力。从行业定义、市场规模、竞争格局、技术趋势、政策环境、投融资动态等多维度进行**广覆盖、深挖掘**的全面调研，输出结构化的行业研究报告。

---

## 1) Overview

### 触发方式（Triggers）
- "帮我调研一下 XX 行业/赛道"
- "做一份 XX 领域的行业研究报告"
- "分析 XX 行业的市场规模和竞争格局"
- "XX 行业有哪些主要玩家？各自优劣势是什么？"
- "XX 赛道的投融资情况和发展趋势"
- "我想进入 XX 市场，帮我做个行业分析"

### 输入（Inputs）
- **必选**：行业/赛道名称或关键词
- **可选**：
  - 地理范围（全球/中国/北美/欧洲等）
  - 时间范围（默认：近 1-3 年）
  - 研究深度（概览/标准/深度）
  - 关注重点（市场规模/竞争格局/技术趋势/政策/投融资）
  - 对标公司（指定分析某些公司）

### 核心输出（Deliverables）
- **飞书云文档行业研究报告**：结构标准、数据详实、可追溯

### 研究原则
- **广度优先**：多维度、多来源、多视角全面覆盖
- **深度挖掘**：核心问题深入分析，不浮于表面
- **数据可信**：核心数据必须标注来源
- **时效性**：优先最新数据，旧数据标注年份

## 3) Module A — 广度检索（主 Agent 执行）

> **核心原则**：多工具、多维度、多来源，确保覆盖全面。

### A1. 检索工具组合

| 工具 | 用途 | 适用场景 |
|------|------|---------|
| `web_search` | 广泛检索公开信息 | 市场规模、新闻动态、公开报告 |
| `fetch` | 获取特定网页/文档全文 | 深入阅读权威报告、招股书、白皮书 |
| `browser` | 交互式网页浏览 | 动态页面、需要登录/操作的数据源 |
| `knowledge_answer` | 检索内部知识库 | 公司内部相关信息、历史研究 |
| `feishu_doc_search` | 检索飞书云文档 | 内部已有的行业研究、分析报告 |

## 4) Module B — 深度挖掘（主 Agent 执行）

> **核心原则**：对关键信息源进行深度阅读和分析，不止于摘要。

### B1. 权威报告深度阅读

**使用 `fetch` 工具获取全文**：

```
fetch(
  url: "https://example.com/industry-report-2024.pdf",
  prompt: "提取该行业报告的核心数据：市场规模、增速、主要玩家份额、技术趋势、政策影响"
)
```

**优先深挖的报告类型**：
- 咨询公司行业报告（麦肯锡、BCG、艾瑞、IDC）
- 上市公司招股书/年报
- 行业协会白皮书
- 政府部门统计报告

**写入 `draft.md`**（深度分析章节）：
```markdown
## 深度报告摘要

- ...

- 市场规模：XX亿元
- 增速预测：CAGR X%
- 头部份额：CR5 = X%

- ...
```

## 10) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 广度不足 | 检索维度不全 | 严格执行 6 大维度检索 |
| 深度不够 | 只看摘要不读全文 | 用 fetch 获取核心报告全文 |
| 数据冲突 | 单一来源 | 交叉验证，差异需说明 |
