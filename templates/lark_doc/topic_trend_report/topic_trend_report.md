# SKILL: topic_trend_report

根据用户指定的**主题**和**时间范围**，自动检索权威动态，生成一份**飞书云文档动态报告**，按类别聚类，每条新闻含 intro / description / 索引链接。强调**时间筛选 + 权威优先 + 去重**。

---

## 1) Overview

### 触发方式（Triggers）
- "给我 {主题} 的日报/周报"
- "帮我整理 {日期/时间段} 的 {主题} 动态"
- "从 {日期} 到 {日期} 的 AI 新闻"
- "最近一周的区块链动态"
- "2024年1月的游戏行业大事"

### 输入（Inputs）

| 参数 | 是否必选 | 说明 | 示例 |
|------|---------|------|------|
| **主题** | 必选 | 报告聚焦的领域/关键词 | AI、区块链、游戏、新能源、半导体 |
| **时间范围** | 可选 | 支持多种格式（默认：今日） | 见下方时间格式说明 |
| **关注子领域** | 可选 | 主题下的细分方向 | LLM、多模态、Agent |
| **关注公司/机构** | 可选 | 重点追踪的组织 | OpenAI、Anthropic、Google |

#### 时间范围格式支持

| 格式类型 | 示例 | 解析结果 |
|---------|------|---------|
| 单日 | "今天"、"昨天"、"2024-01-15" | 当日 00:00-23:59 |
| 日期范围 | "1月10日到1月15日"、"2024-01-10 ~ 2024-01-15" | 起始日 00:00 ~ 结束日 23:59 |
| 相对时间 | "最近3天"、"过去一周"、"本月" | 自动计算时间窗口 |
| 年月 | "2024年1月"、"上个月" | 整月范围 |

### 核心输出（Deliverables）
- **飞书云文档主题日报**：按类别聚类、结构清晰、可追溯

### 质量标准
- **时间准确**：所有条目必须验证发布时间在指定窗口内
- **权威优先**：官方源 > 权威媒体 > 社区讨论
- **去重**：同一事件多来源只保留主条目

---

## 2) 关键流程

### Phase 1: 参数解析与确认

```
用户输入 → 解析主题 + 时间范围 → 确认参数
```

**Agent 行为**：
1. 从用户输入中提取：主题关键词、时间表述、可选的子领域/公司
2. 将时间表述转换为精确的时间窗口（UTC+8）
3. 向用户确认理解是否正确（如有歧义）

**示例**：
```
用户: "帮我整理上周的 AI 大模型动态"
Agent 理解:
- 主题: AI 大模型
- 时间窗口: 2024-01-08 00:00 ~ 2024-01-14 23:59 (UTC+8)
- 关注方向: LLM、大语言模型
```

---

### Phase 2: 多源检索

> **核心原则**：根据主题选择合适的信息源 + 时间过滤

#### 2.1 信息源选择（按主题动态调整）

| 主题类型 | 优先信息源 | 补充信息源 |
|---------|-----------|-----------|
| AI/科技 | 官方博客、arXiv、GitHub | Hacker News、Product Hunt、Twitter |
| 区块链/Web3 | 项目官网、CoinDesk、The Block | Twitter、Discord、GitHub |
| 游戏 | 官方公告、IGN、Polygon | Steam、Reddit、YouTube |
| 金融 | 官方公告、Reuters、Bloomberg | SEC Filings、财报 |
| 通用 | Google News、权威媒体 | 社交媒体、行业论坛 |

#### 2.2 检索策略

**web_search 查询模板**：
```
"{主题关键词}" (发布 OR 更新 OR 公告) {时间限定词}
site:{权威域名} "{主题}" {日期}
"{关注公司}" "{主题}" (release OR announcement) {时间}
```

**fetch 深入获取**：
```
fetch(
  url: "{搜索结果中的重要链接}",
  prompt: "提取发布时间、标题、核心内容、关键数据"
)
```

---

### Phase 3: 时间核验与筛选

**核验规则**：
- 每条候选必须确认发布时间
- 时间不在指定窗口：**直接剔除**
- 无法确认时间：**不纳入**

**核验方式**：
1. 页面内查找发布时间/更新时间
2. URL 中的日期信息
3. 页面 meta 信息

---

### Phase 4: 去重与分类

**去重规则**：
- 同一事件多来源：保留原始来源，其他作为补充链接
- 同一主体多更新：合并为一条

**通用分类框架**（按主题调整）：
1. **官方发布 & 重大更新** - 来自官方的第一手信息
2. **产品 & 技术** - 新产品、技术突破、版本更新
3. **研究 & 论文** - 学术成果、研究报告
4. **开源 & 工具** - 开源项目、开发者工具
5. **行业 & 市场** - 投融资、合作、市场动态
6. **观点 & 分析** - 深度分析、行业评论

---

### Phase 5: 生成报告

调用 Writer Agent 生成最终的飞书云文档。

## 5) 质量检查清单

- [ ] 主题关键词已明确

- [ ] 歧义已与用户确认

### 信息检索
- [ ] 已选择合适的信息源组合
- [ ] 官方/权威源已优先检索
- [ ] 补充源已检索

### 时间核验
- [ ] 所有条目已核验发布时间
- [ ] 时间窗口外内容已剔除
- [ ] 无法确认时间的内容已排除

### 内容整理
- [ ] 已完成去重
- [ ] 已按类别分类
- [ ] 每条含完整信息（标题/简介/详情/链接/时间）

### 文档输出
- [ ] 结构完整（要点/分类/附录）
- [ ] 核心要点突出
- [ ] 已转换为飞书云文档

## 6) 常见问题处理

| 问题 | 处理方式 |
|------|---------|
| 时间范围模糊 | 向用户确认具体日期 |
| 主题过于宽泛 | 建议用户缩小范围或指定子领域 |
| 某时间段无内容 | 如实告知，建议扩大时间范围 |
| 信息源受限 | 说明限制，使用替代源 |
| 内容过多 | 按重要性筛选，保留 Top N |
