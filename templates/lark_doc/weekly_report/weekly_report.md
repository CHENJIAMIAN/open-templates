# SKILL: weekly_work_report

基于飞书生态的**每周周报**自动生成 Skill：从当周**日程、任务、妙记、云文档**中提取工作内容，使用 `knowledge_answer` 兜底补全遗漏信息，最终调用 `doc_agent (writer)` 生成结构化的飞书云文档周报。

---

## 1) Overview

### 触发方式（Triggers）
- "帮我生成本周周报"
- "总结一下我这周的工作"
- "把这周的日程、任务、会议整理成周报"
- "生成 YYYY-MM-DD 到 YYYY-MM-DD 的周报"

### 时间窗口（Week Window）
- **默认**：本周（周一 00:00:00 至周日 23:59:59），按用户时区
- **自定义**：用户可指定具体日期范围
- 周报必须标注：`统计区间：YYYY-MM-DD（周一）~ YYYY-MM-DD（周日）`

### 核心输出（Deliverables）
- **飞书云文档周报**：结构标准、内容翔实、可追溯

### 质量标准
- **完整性**：日程、任务、妙记、云文档四类来源均覆盖
- **洞察优先**：不做流水账，每类来源产出 insights
- **时间正确性**：严格过滤到当周窗口

## 2) 任务分解示例（todo_write）

主 Agent 收到周报请求后，使用 `todo_write` 分解任务：

```json
{
  "todos": [
    {"content": "获取本周日程数据并分析", "status": "in_progress"},
    {"content": "获取本周任务数据并分析", "status": "pending"},
    {"content": "获取本周妙记（会议纪要）并提取要点", "status": "pending"},
    {"content": "获取本周创建/编辑的云文档", "status": "pending"},
    {"content": "使用 knowledge_answer 兜底补全遗漏信息", "status": "pending"},
    {"content": "汇总分析结果，调用 writer agent 生成周报云文档", "status": "pending"}
  ]
}
```

## 3) Module A — 数据获取（主 Agent 执行）

> **核心原则**：使用 MCP 工具获取四类数据源，最后用 `knowledge_answer` 兜底。

### A1. 获取本周日程

**工具**：`mcp__ms_official_calendar__*` 或相关日程 MCP

**获取内容**：
- 事件标题、开始/结束时间
- 参与人
- 会议描述/备注

**时间过滤**：仅保留落在周窗口（周一至周日）的事件

**分析维度**：
- 会议数量与时长统计
- 按项目/主题聚类
- 高频协作对象

**写入 `draft.md`**：
```markdown
## 日程数据

### 原始数据
| 日期 | 时间 | 事件 | 参与人 |
|------|------|------|--------|
| 01-13 | 10:00-11:00 | 项目A周会 | @张三, @李四 |
| ... | ... | ... | ... |

### 洞察
- 本周共 N 场会议，总时长约 X 小时
- Top 3 会议主题：项目A、项目B、技术评审
- 高频协作：@张三（5次）、@李四（3次）
```

---

### A2. 获取本周任务

**工具**：`mcp__ms_official_aily_lark_task__*` 或相关任务 MCP

**获取内容**：
- 任务标题、负责人、状态（待办/进行中/已完成）
- 截止时间、优先级
- 更新记录

**过滤规则**：
- 本周创建的任务
- 本周完成的任务
- 本周有状态变更的任务

**分析维度**：
- 完成数 / 进行中数 / 阻塞数
- 关键完成项
- 阻塞原因与待解决事项

**写入 `draft.md`**：
```markdown
## 任务数据

### 本周完成
- [x] 任务1：完成XX功能开发
- [x] 任务2：提交XX文档

### 进行中
- [ ] 任务3：XX优化（预计下周完成）

### 阻塞
- [ ] 任务4：等待XX审批 —— 阻塞原因：依赖外部团队

### 洞察
- 本周完成 N 项任务，完成率 X%
- 阻塞事项需关注：任务4（影响后续排期）
```

---

### A3. 获取本周妙记（会议纪要）

**工具**：`mcp__ms_official_aily_mcp_minute__*` 或相关妙记 MCP

**获取内容**：
- 会议标题、时间
- 会议摘要/纪要内容
- 决策事项、行动项

**分析维度**：
- 关键决策清单
- 待办行动项（Action Items）
- 重要讨论要点

**写入 `draft.md`**：
```markdown
## 妙记数据

### 重要会议纪要
#### 1. 项目A周会（01-13）
- **决策**：确定采用方案B
- **行动项**：@张三 完成原型设计（01-17前）
- **要点**：讨论了技术选型，达成一致

#### 2. 技术评审会（01-15）
- **决策**：代码review流程优化
- **行动项**：@李四 更新开发规范文档

### 洞察
- 本周 N 场会议有纪要记录
- 关键决策 X 项，待跟进行动项 Y 项
```

---

### A4. 获取本周云文档活动

**工具**：`feishu_doc_search` + `feishu_doc_list`

**检索方式**：
```
feishu_doc_search(
  query: "",
  filters: {
    owner_name: "{当前用户}",
    create_time: "[YYYY-MM-DD, YYYY-MM-DD]"  // 本周范围
  }
)
```

**获取内容**：
- 文档标题、链接
- 创建/更新时间
- 文档类型（文档/表格/幻灯片）

**分析维度**：
- 本周创建的文档（产出）
- 本周编辑的文档（贡献）
- 文档分类（需求文档/技术方案/会议纪要/其他）

**写入 `draft.md`**：
```markdown
## 云文档活动

### 本周创建
| 文档标题 | 类型 | 链接 |
|---------|------|------|
| XX技术方案 | 文档 | [链接](url) |
| XX数据分析 | 表格 | [链接](url) |

### 本周编辑
| 文档标题 | 更新内容 |
|---------|---------|
| 项目A PRD | 补充用户场景 |

### 洞察
- 本周产出 N 篇文档
- 主要产出类型：技术方案（X篇）、需求文档（Y篇）
```

---

### A5. Knowledge Answer 兜底

**工具**：`knowledge_answer`

**查询策略**（在前4步完成后执行）：
```
knowledge_answer(
  query_list: ["总结我本周（YYYY-MM-DD到YYYY-MM-DD）的主要工作内容和产出"],
  explanation: "兜底补全可能遗漏的本周工作信息"
)
```

**使用规则**：
- 作为**补充来源**，不替代前4步的事实数据
- 若 knowledge_answer 返回的信息与前4步冲突，以 MCP 数据为准
- 若发现前4步遗漏的重要信息，补充到 `draft.md`

**写入 `draft.md`**：
```markdown
## Knowledge Answer 补充

### 补充信息
- [补充] 参与了XX项目的紧急修复（未在日程中体现）
- [补充] 完成了XX文档的评审（未在任务中体现）

### 与事实源对照
- 一致：日程、任务、妙记信息与 knowledge_answer 基本吻合
- 补充：发现 X 项遗漏，已补入对应章节
```

- 主线2：XX技术优化

## 5) Module C — 调用 Writer Agent 生成周报

### 任务分发

主 Agent 完成数据收集后，调用 `task` 工具分发给 writer agent：

```json
{
  "description": "生成本周周报云文档",
  "prompt": "基于 /home/workspace/draft.md 中的周报素材，生成一份飞书云文档周报。\n\n要求：\n1. 严格按照周报模板结构\n2. 统计区间：YYYY-MM-DD ~ YYYY-MM-DD\n3. 突出洞察和结论，避免流水账\n4. 所有数据来源于 draft.md，不要编造\n\n周报模板见下方。",
  "subagent_type": "writer"
}
```

## 9) 常见失败模式与修复

| 失败模式 | 原因 | 修复方法 |
|---------|------|---------|
| 数据缺失 | MCP 工具调用失败 | 使用 knowledge_answer 兜底 |
| 内容流水账 | 缺少分析 | 每类数据必须有"洞察"小节 |
| 时间范围错误 | 未正确设置过滤条件 | 检查日期参数，确保是周一至周日 |
| 周报结构混乱 | Writer 未遵循模板 | 在 prompt 中明确要求遵循模板 |
| 信息重复 | draft.md 有冗余 | 去重后再调用 writer |
